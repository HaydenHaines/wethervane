# Decisions Log

Key architectural and methodological decisions for the US Political Covariation Model. Each decision records the date, what was decided, why, and a link to a formal ADR where applicable.

For full context on any decision, see the linked ADR in `docs/adr/`.

---

| Date | Decision | Rationale | ADR |
|------|----------|-----------|-----|
| 2026-03-10 | Hybrid Python + R + Stan stack: Python for data assembly and community detection, R for MRP and Bayesian modeling, Stan models as shared artifacts callable from both. | Best tool per task. Community detection libraries (leidenalg, graph-tool, scikit-learn NMF) are Python. The closest existing election model (Economist 2020) and the best MRP pipeline (ccesMRPprep) are R. Stan is the natural choice for custom Bayesian models and is callable from both ecosystems via CmdStanPy/CmdStanR. Forcing everything into one language would mean using weaker tools for at least one stage. | [ADR-001](adr/001-hybrid-python-r-stan.md) |
| 2026-03-10 | Community types as soft assignments: each county is a mixture of types (topic model / NMF framing) rather than belonging to exactly one type. | Counties are heterogeneous. A county with 40% evangelical rural, 30% military retiree, and 30% suburban professional population should not be forced into a single category. Soft assignment enables spectral unmixing for poll decomposition and produces more accurate county-level predictions for heterogeneous places. The NMF / topic-model analogy is well-established (counties as "documents," community types as "topics," demographic/network features as "words"). | [ADR-002](adr/002-community-type-soft-assignment.md) |
| 2026-03-10 | FL+GA+AL as proof-of-concept geography (226 counties). | This region provides community diversity (Cuban, Puerto Rican, retiree, military, evangelical, Black Belt, suburban, university), political heterogeneity (Florida and Georgia have recent flip history, Alabama provides a safe-state anchor), cross-border community testing (North FL / South GA / South AL share community structure across state lines), and good data availability (Florida voter files are among the most accessible nationally). | [ADR-003](adr/003-fl-ga-al-proof-of-concept.md) |
| 2026-03-10 | Dual-output model: jointly model vote share and turnout at the community-type level. | Enables decomposition of election-to-election shifts into persuasion (vote share change at constant turnout), mobilization (turnout change at constant vote share), and composition (population/registration change). This decomposition is a key interpretive output of the model. Joint modeling also respects the empirical correlation between turnout and vote share within community types (Assumption A007). | -- |
| 2026-03-10 | Two-stage separation: discover community types from non-political data first, then validate political relevance second. | Prevents circularity. If community types are defined by political outcomes, the model's "prediction" is partially tautological. Stage 1 uses demographics, religion, commuting, migration, and social network data only. Stage 2 checks whether the discovered types exhibit meaningful political covariance. This separation is also a falsifiability criterion: if non-political community structure does not predict political covariance, the core thesis is wrong. | -- |
| 2026-03-10 | Use the Economist 2020 presidential model as the primary architectural template for the propagation stage. | The Economist model (Linzer, Lauderdale, and Silver -- open source, Stan-based) is the closest existing implementation to what the propagation stage needs: poll aggregation, state-level correlation structure, prior specification from fundamentals. The key extension is replacing their demographic correlation with community-type covariance. Starting from a working model reduces risk compared to building from scratch. | -- |
| 2026-03-10 | Factor-structured covariance (low-rank factor model) rather than full covariance matrix for community types. | With K community types and only T election cycles (where K >> T), a full K x K covariance matrix is unidentifiable. A factor model with F << K factors (Assumption A006 suggests F = 3-5) reduces the parameter count from O(K^2) to O(K * F) and regularizes the covariance estimate. This is standard practice in financial portfolio models with similar dimensionality constraints. | -- |
| 2026-03-10 | Voter stability as a strong prior in the Bayesian model. | Supported by Gelman et al. 2016 ("The Mythical Swing Voter"), Kalla & Broockman 2018 (minimal persuasion effects), and ANES panel data showing high individual vote stability. The prior encodes the belief that large type-level swings are unlikely and that most apparent movement comes from differential turnout and composition change. This prior can be relaxed if evidence warrants (see Assumption A001). | -- |
