# Data Verification Findings — 2026-04-08

## Critical: Worktree vs. Main Project Data

The worktree at `.worktrees/feat-backtest-harness/` has its own `data/` subtree. The
`.gitignore` excludes all the data-heavy directories (`data/assembled/`,
`data/communities/`, `data/shifts/`, etc.), so the worktree only has the
smaller directories that *are* tracked: `data/models/`, `data/polls/`,
`data/fundamentals/`, `data/config/`, etc.

**Consequence for Tasks 3–5:** Any code that resolves paths relative to
`PROJECT_ROOT` (which all production modules do) will look in
`<worktree>/data/` when run from the worktree. `data/assembled/` and
`data/communities/` don't exist there. Two options:

1. Accept an explicit `data_root: Path` parameter in backtest modules (recommended).
2. Point at the main project's data dir at
   `/Users/haydenhaines/Projects/ClaudeProjects/US-political-covariation-model/data/`.

## 1. Production Pipeline — Type Score Loading

**Key importable functions in `src/prediction/predict_2026_types.py`:**

- `run()` — the main entry point; not intended to be called with arguments. All
  parameters are module-level constants loaded from `data/config/prediction_params.json`.
- `_load_type_data()` — private; returns `(county_fips, type_scores, type_covariance, type_priors)`.
- `_load_county_metadata()` — private.
- `_load_county_votes()` — private.
- `_load_polls()` — private.

There are **no public importable functions** beyond `run()`. The module is not
designed to be called with custom data paths or years. Tasks 3–5 should call
`src.prediction.county_priors` and `src.prediction.forecast_engine` directly,
not `predict_2026_types.run()`.

**Type scores file:** `data/communities/type_assignments.parquet`
- **This file does NOT currently exist** (not in worktree, not in main project data).
- The existing file `data/communities/county_type_assignments.parquet` has a
  different schema (`community_id`, `type_weight_0..6`, `dominant_type_id`, `j`)
  and covers only 10 rows (communities, not counties). It is not the file the
  production pipeline expects.
- The expected future file uses `*_score` suffix columns:
  ```
  county_fips, <type_0_score>, <type_1_score>, ..., <type_J_score>
  ```
  where score columns satisfy `c.endswith("_score")`.

**FIPS column:** `county_fips` (zero-padded to 5 digits)

**State column:** Derived from `county_fips[:2]` via a state FIPS lookup. Not
stored in `type_assignments.parquet`. Available in `assembled/medsl_county_presidential_{year}.parquet` as `state_abbr`.

**Vote weight column:** `totalvotes` — loaded from
`data/raw/medsl_county_presidential_2024.parquet` (note: this is in `data/raw/medsl/`,
not `data/assembled/`; the assembled files live in `data/assembled/`). Falls back
to `1.0` per county if file is unavailable.

**`load_county_priors_with_ridge()` signature:**
```python
load_county_priors_with_ridge(
    county_fips: list[str],
    ridge_priors_path: Path | None = None,  # defaults to data/models/ridge_model/ridge_county_priors.parquet
    assembled_dir: Path | None = None,      # defaults to data/assembled/
) -> np.ndarray  # shape (N,), one prior Dem share per county
```

**`load_county_priors_with_ridge()` returns:**
- An `ndarray` of shape `(N,)` with prior Dem share per county.
- Tries Ridge model predictions first (`ridge_pred_dem_share` column from
  `data/models/ridge_model/ridge_county_priors.parquet`).
- Falls back to most-recent historical presidential Dem share from
  `data/assembled/medsl_county_presidential_{year}.parquet` (reads the
  `pres_dem_share_{year}` column).
- Falls back to 0.45 if no historical data.

## 5. MEDSL County Actuals

### Presidential — assembled parquets

**Files:** `data/assembled/medsl_county_presidential_{year}.parquet`

Available years: 2000, 2004, 2008, 2012, 2016, 2020, 2024

**Schema (consistent across all years):**
```
county_fips    — str, zero-padded 5-digit FIPS
state_abbr     — str, 2-letter abbreviation
pres_dem_{year}  — float, Democratic votes
pres_rep_{year}  — float, Republican votes
pres_total_{year} — float, total votes
pres_dem_share_{year} — float, two-party Democratic share [0,1]
```

**Shape:** 293 rows — **FL, GA, AL only** (pilot states). Not national.

**National raw source available:** `data/raw/medsl/county_presidential_2000_2024.tab`
(94,151 rows, all 51 states/DC, years 2000–2024, columns:
`state, county_name, year, state_po, county_fips, office, candidate, party, candidatevotes, totalvotes, version, mode`)

The raw tab file has national coverage and could be used to assemble national
presidential actuals if needed.

### Governor — assembled parquets

**Files:**

**Algara schema:**
```
county_fips, state_abbr, gov_dem_{year}, gov_rep_{year}, gov_total_{year}, gov_dem_share_{year}
```

**MEDSL 2022 governor schema:**

**Shape:** 293 rows — **FL, GA, AL only** for all governor files.
